# 최적화 서비스 비교 (Issue #25)

비방향 그래프의 각 간선에 방향을 부여하여 최적의 방향 그래프를 찾는 3가지 서비스입니다.
모두 동일한 인터페이스(`WeightedOptimizationService`)를 구현하며, 입력(`WeightedGraph`)과 출력(`list[tuple[int, int]]`)이 동일합니다.

---

## 1. PolynomialOptimizationService (MR2S / Small World)

**엔드포인트:** `POST /api/v1/mr2s`

**목적:** QUBO 다항식을 구성하고 시뮬레이티드 어닐링 샘플러로 풀어 최적 간선 방향을 결정

### 처리 흐름

1. `WeightedGraph`에서 정점 집합과 인접 리스트를 추출
2. 등록된 `PolynomialGenerator` 목록을 순회하며 각각 다항식을 생성
   - **FlowConservationPolynomialGenerator**: 각 정점에서 들어오는 간선과 나가는 간선의 균형(흐름 보존)을 유도하는 페널티 다항식
   - **MinimizeSumOfApspPolynomialGenerator**: n-hop 경로 수의 총합을 최대화(= -1을 곱해 최소화 문제로 변환)하는 다항식. `SmallWorldSpec`에 정의된 hop 수(예: 2-hop, 3-hop)와 가중치에 따라 생성
3. 모든 다항식을 합산하여 하나의 `BinaryPolynomial` (QUBO 식)을 완성
4. `make_quadratic`로 2차식 BQM으로 변환 후, D-Wave `SimulatedAnnealingSampler`로 `num_reads`(기본 100)회 샘플링
5. 샘플 중 실제 APSP 거리가 가장 짧은 것을 선택 (비강연결 시 `float('inf')` 처리, 강연결 해 우선)
6. 각 이진 변수 `e_i_j`의 값(0 또는 1)에 따라 간선 방향을 결정하여 반환

### 변수 규칙

- 간선 `(i, j)` (i < j)에 대해 이진 변수 `e_i_j`를 생성
- `e_i_j = 0` → 방향: i → j
- `e_i_j = 1` → 방향: j → i

---

## 2. NaotoService (Raw Simulated Annealing)

**엔드포인트:** `POST /api/v1/raw-sa`

**목적:** QUBO를 사용하지 않고, 직접 그래프 메트릭을 평가하며 시뮬레이티드 어닐링으로 최적 간선 방향을 탐색

### 처리 흐름

1. `WeightedGraph`의 정점을 0-based 인덱스로 재매핑 (내부적으로 `optimize_edge_orientations`가 `range(n)` 기반으로 동작하기 때문)
2. 각 간선에 대해 방향(+1: min→max, -1: max→min)을 랜덤으로 초기화
3. 시뮬레이티드 어닐링 루프 실행 (기본 1200 스텝):
   - 랜덤으로 간선 하나를 선택하여 방향을 반전
   - 변경된 방향으로 NetworkX `DiGraph`를 구성
   - 4가지 메트릭을 계산하여 스코어를 산출:
     - **도달 불가능 쌍 수** (× 10000, 최우선)
     - **평균 최단 거리** (× 1.0)
     - **간선 부하 제곱합** (× 0.06) — 특정 간선에 트래픽이 집중되는 것을 억제
     - **최대 간선 부하** (× 0.8) — 병목 간선을 억제
   - 스코어가 개선되면 수락, 악화되면 온도에 따른 확률로 수락/거부
   - 온도: 3.0 → 0.02로 지수적 감소
4. 최적 스코어의 방향 배정을 선택
5. 0-based 인덱스를 원래 정점 ID로 역매핑하여 반환

### MR2S와의 차이점

| 항목 | MR2S | Raw SA |
|------|------|--------|
| 비용 함수 | QUBO 다항식 (흐름 보존 + n-hop 경로) | 직접 그래프 메트릭 (APSP + 부하 분산) |
| 풀이 방법 | D-Wave SA 샘플러 (BQM) | 직접 구현한 SA |
| 가중치 사용 | 간선 가중치를 다항식에 반영 | 간선 가중치 미사용 (무가중 그래프로 처리) |
| 평가 기준 | 다항식 에너지 → APSP로 검증 | 도달성 + 평균거리 + 부하분산 복합 스코어 |

## 3. BruteForceService (완전 탐색)

**엔드포인트:** `POST /api/v1/brute-force`

**목적:** QUBO 없이 모든 간선 방향 조합을 전수 탐색하여 APSP 총합이 최소인 전역 최적해를 보장

### 처리 흐름

1. `WeightedGraph`에서 정점 집합과 간선 목록을 추출
2. 모든 가능한 방향 조합을 열거 (2^E개)
   - 각 간선마다 0 또는 1을 배정: `0` → `vertices[0] → vertices[1]`, `1` → `vertices[1] → vertices[0]`
3. 각 조합에 대해 `calculate_total_apsp_distance`로 방향 그래프의 APSP 총합을 직접 계산
   - 비강연결 그래프는 `float('inf')` 처리 (강연결 해 우선, 없으면 비강연결이라도 반환)
4. 가장 낮은 APSP 총합을 가진 조합을 선택하여 반환

### 시간 복잡도

- **O(2^E × V × (V + E))** (E: 간선 수, V: 정점 수 — 각 조합마다 APSP 계산)
- 간선이 약 20개를 초과하면 실용적이지 않음
- 주 용도: 소규모 그래프에서 다른 서비스의 결과가 전역 최적해에 얼마나 가까운지 검증

## 공통 응답 구조 (ResponseDto)

세 서비스 모두 동일한 후처리를 거쳐 응답합니다:

1. 최적화 결과인 `list[tuple[int, int]]` (방향 간선 목록)을 받음
2. `calculate_total_apsp_distance`로 **방향 그래프의 APSP 총합** 계산
3. 같은 간선 목록을 **비방향 그래프로 변환**하여 APSP 총합 계산 (비교 기준)
4. 두 점수와 간선 목록을 `ResponseDto`로 반환

```json
{
  "edges": [{"_from": 0, "to": 1}, {"_from": 2, "to": 1}],
  "optimized_graph_score": 42.0,
  "bidirectional_graph_score": 30.0
}
```

### 비강연결 그래프 처리

| 서비스 | 비강연결 처리 방식 |
|--------|-------------------|
| MR2S | APSP가 -1이면 `float('inf')` 처리, 강연결 해 우선 선택 |
| Raw SA | 도달 불가능 쌍 수 × 10000 페널티로 강연결을 강하게 유도 |
| Brute Force | APSP가 -1이면 `float('inf')` 처리, 강연결 해 우선 선택 |

## 예상 결과 비교

### 테스트 입력: 삼각형 그래프 (3정점, 3간선)

```
정점: {0, 1, 2}
간선: 0-1 (weight=1), 1-2 (weight=1), 0-2 (weight=1)
```

```json
{
  "edges": [
    {"vertices": [0, 1], "weight": 1},
    {"vertices": [1, 2], "weight": 1},
    {"vertices": [0, 2], "weight": 1}
  ]
}
```

가능한 방향 조합: 2^3 = 8가지

| # | 0-1 | 1-2 | 0-2 | 강연결 | APSP 총합 |
|---|-----|-----|-----|--------|-----------|
| 0 | 0→1 | 1→2 | 0→2 | X | - |
| 1 | 0→1 | 1→2 | 2→0 | O | 6 (순환: 0→1→2→0) |
| 2 | 0→1 | 2→1 | 0→2 | O | 6 (순환: 0→2→1... 아닌 경우 확인 필요) |
| 3 | 0→1 | 2→1 | 2→0 | X | - |
| 4 | 1→0 | 1→2 | 0→2 | X | - |
| 5 | 1→0 | 1→2 | 2→0 | O | 6 (순환: 1→2→0→1... 아닌 경우 확인 필요) |
| 6 | 1→0 | 2→1 | 0→2 | O | 6 (순환: 0→2→1→0) |
| 7 | 1→0 | 2→1 | 2→0 | X | - |

삼각형에서 강연결이 되려면 3개 간선이 같은 방향으로 순환해야 합니다.
강연결 조합은 정확히 2가지 (시계/반시계)이며 둘 다 APSP 총합 = **6** 입니다.

**예상 결과:**

| 서비스 | optimized_graph_score | bidirectional_graph_score | 비고 |
|--------|----------------------|--------------------------|------|
| MR2S | 6 | 6 | QUBO SA로 탐색, 최적해를 찾을 확률 높음 |
| Raw SA | 6 | 6 | 직접 SA로 탐색, 최적해 도달 가능 |
| Brute Force | 6 | 6 | 전역 최적해 보장 |

### 테스트 입력: 사각형 그래프 (4정점, 4간선)

```
정점: {0, 1, 2, 3}
간선: 0-1, 1-2, 2-3, 0-3 (모두 weight=1)
```

```json
{
  "edges": [
    {"vertices": [0, 1], "weight": 1},
    {"vertices": [1, 2], "weight": 1},
    {"vertices": [2, 3], "weight": 1},
    {"vertices": [0, 3], "weight": 1}
  ]
}
```

가능한 방향 조합: 2^4 = 16가지

사각형에서 강연결 순환은 2가지 (시계/반시계)이며 APSP 총합 = **20** (대각 정점 거리 = 2)

**예상 결과:**

| 서비스 | optimized_graph_score | bidirectional_graph_score | 비고 |
|--------|----------------------|--------------------------|------|
| MR2S | 20 | 12 | 사각형은 순환 구조라 방향 그래프가 불리 |
| Raw SA | 20 | 12 | 동일한 최적해 예상 |
| Brute Force | 20 | 12 | 전역 최적해 보장 |

### 테스트 입력: 대각선 있는 사각형 (4정점, 5간선)

```
정점: {0, 1, 2, 3}
간선: 0-1, 1-2, 2-3, 0-3, 1-3 (모두 weight=1)
```

```json
{
  "edges": [
    {"vertices": [0, 1], "weight": 1},
    {"vertices": [1, 2], "weight": 1},
    {"vertices": [2, 3], "weight": 1},
    {"vertices": [0, 3], "weight": 1},
    {"vertices": [1, 3], "weight": 1}
  ]
}
```

가능한 방향 조합: 2^5 = 32가지

대각선이 추가되면 강연결 해가 더 많아지고, APSP를 더 줄일 수 있습니다.

**예상 결과:**

| 서비스 | optimized_graph_score | bidirectional_graph_score | 비고 |
|--------|----------------------|--------------------------|------|
| MR2S | 14~16 | 10 | 대각선 덕에 최단 경로 개선, SA 품질에 따라 차이 |
| Raw SA | 14~16 | 10 | 부하 분산도 고려하므로 결과가 다를 수 있음 |
| Brute Force | 14 | 10 | 전역 최적해 보장 |

> **참고:** MR2S와 Raw SA는 확률적 알고리즘이므로 실행마다 결과가 달라질 수 있습니다.
> Brute Force만 항상 동일한 전역 최적해를 반환합니다.
> 위 수치는 수작업 계산 기반 예상치이며, 실제 결과와 다를 수 있습니다.
